# 修仙挂机游戏 - 随机数系统说明文档

## 概述

本文档详细说明了游戏中所有涉及随机数的场景，以及每个随机数对玩家的影响方向。

**核心原则：**
- **我方（玩家/韵灵/生灵）：随机数越小越有利**
- **敌方（怪物/敌人）：随机数越大对敌方越有利（对玩家越不利）**

---

## 二、战斗结束掉落随机数（越小越好）

| 序号 | 场景 | 判定条件 | 掉落概率 |
|------|------|----------|----------|
| 15 | 灵石掉落 | `random < 0.01` | 1% |
| 16 | 仙石掉落 | `random < 0.0002` | 0.02% |
| 17 | 法则碎片掉落 | `random < 0.05` | 5% |
| 18 | 仙草掉落判定 | `random < 0.01` | 1% |
| 19 | 仙草品阶计算 | `logR = -log10(r)` | r越小品阶越高 |
| 20 | 时间结晶掉落 | `random < 1/6000` | 1/6000 |

### 仙草品阶计算公式

```
品阶 = floor(怪物等级 × log10(1.035) - log10(random))
```

- random 越小，log10(random) 越大（负数），品阶越高
- random = 0 时返回 0（边界处理）

## 五、世界探索系统（双方使用相同随机数）

### 5.1 核心原理

世界探索系统中，玩家单位和敌方单位使用**相同的随机数范围和计算公式**：

```javascript
const rand = Math.random();
// 双方使用相同的 rand
```

**效果说明：**
- 我方：随机数越小，效果越强（对我方有利）
- 敌方：随机数越大，效果越强（对敌方有利，对玩家不利）

### 5.2 技能效果范围（双方相同）

| 技能名称 | 效果 | 范围 | 计算公式 |
|----------|------|------|----------|
| 灵泉润枯法 | 回血百分比 | 50%~200% | `50 + rand * 151` |
| 碎虚绝魂击 | 阈值百分比 | 20%~60% | `20 + rand * 41` |
| 蚀血噬魂打 | 伤害百分比 | 20%~60% | `20 + rand * 41` |
| 千影连星刺 | 攻击次数 | 2~6次 | `2 + rand * 5` |
| 戮绝残血式 | 伤害百分比 | 200%~600% | `200 + rand * 401` |
| 血元反哺术 | 回血百分比 | 50%~200% | `50 + rand * 151` |
| 万域镇魔波 | 递减百分比 | 0%~50% | `rand * 51` |
| 寒骨封脉击 | 降速值 | 5~20 | `5 + rand * 16` |

### 5.3 实际效果对比

| 随机数 | 我方单位效果 | 敌方单位效果 |
|--------|--------------|--------------|
| 0.1 | 最强（如回血200%） | 最弱（如回血50%） |
| 0.5 | 中等（如回血125%） | 中等（如回血125%） |
| 0.9 | 最弱（如回血50%） | 最强（如回血200%） |

**总结：**
- 我方随机到0.1 → 效果强 → 对玩家有利
- 敌方随机到0.9 → 效果强 → 对敌方有利（对玩家不利）

## 六、长生久视神通原理

### 6.1 核心机制

长生久视神通通过多次模拟战斗，选择最优结果：

```javascript
const simCount = Math.ceil(Math.log(zhutianLevel) / Math.log(1.5)) || 1;
// 等级1: 1次模拟
// 等级2: 2次模拟
// 等级3: 2次模拟
// 等级4: 3次模拟
// 等级5: 4次模拟
```

### 6.2 随机数预生成

每次模拟使用预生成的1000个随机数：

```javascript
const randomArray = Array.from({length: 1000}, () => Math.random());
```

### 6.3 结果选择策略

1. 筛选所有胜利的模拟结果
2. 按回合数排序（用时最短优先）
3. 选择第一个胜利结果

### 6.4 效果分析

由于随机数方向的统一：

- **我方有利事件**（暴击、回血、闪避等）：随机数越小越触发
- **敌方有利事件**（暴击、技能触发等）：随机数越大越触发
- **最终效果**：选择随机数整体较小的模拟结果 = 选择我方运气好、敌方运气差的一次战斗

## 八、修改记录

### 2026-05-10 统一随机数方向

1. **怪物暴击判定**
   - 暴击率≤100%：`random < mCritRate` → `random >= (1 - mCritRate)`
   - 暴击率>100%：`critRoll < fracPart` → `critRoll >= fracPart`
   - 效果：随机数越大，怪物越暴击

2. **怪物技能触发**
   - 有两下子：`random < triggerChance` → `random >= (1 - triggerChance)`
   - 回光返照：`random < triggerChance` → `random >= (1 - triggerChance)`
   - 效果：随机数越大，怪物技能越触发

3. **世界探索技能效果**
   - 双方使用相同的随机数范围和计算公式
   - 我方：随机数越小效果越强
   - 敌方：随机数越大效果越强
