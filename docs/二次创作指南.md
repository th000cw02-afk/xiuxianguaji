# 二次创作指南

Fork 本仓库后，按以下顺序换皮，**无需读完整个 game.js**。

## 1. 最小换皮（约 30 分钟）

编辑 [`app/src/main/assets/theme/config.js`](../app/src/main/assets/theme/config.js)：

```javascript
window.GAME_THEME = {
  productName: '你的游戏名',
  seriesName: '文字修仙',
  realms: ['入门', '小成', '大成', '宗师', '传说'],
  tabs: { cultivate: '修行', battle: '历练', ... }
};
```

编辑 [`app/src/main/assets/theme/contracts.js`](../app/src/main/assets/theme/contracts.js) 中的契约标题与描述。

修改 Android 显示名：[`strings.xml`](../app/src/main/res/values/strings.xml)、[`AndroidManifest.xml`](../app/src/main/AndroidManifest.xml)。

## 2. 题材换皮（约 1~2 天）

| 改什么 | 文件 |
|--------|------|
| 怪物名称 | `game.js` → `monsterData` |
| 地图名称 | `game.js` → `mapNames` |
| 丹药/灵药名 | `game.js` → `pillInfo`、灵药数组 |
| 神通名称 | `game.js` → `defaultPlayer.divineAbilities` |
| 纪事/奇遇文案 | `theme/config.js` → `chroniclePool` |
| 成就名称 | `meta-systems.js` → `ACHIEVEMENT_DEFS` 或后续外置 |

## 3. 数值调整

| 改什么 | 入口 |
|--------|------|
| 升级/突破公式 | `game.js` → `getRequiredCultivation`、`getBreakthroughSuccessRate` |
| 怪物成长 | `game.js` → `startBattle` 怪物属性段 |
| 掉落率 | `game.js` → `addResourceDrop`、`getQiYunProbability` |
| 洞府消耗 | `game.js` → `getSpiritFieldCost`、`upgradeGatheringArray` |

设计参考：[平衡草稿.txt](../app/src/main/assets/平衡草稿.txt)、[游戏说明.txt](../app/src/main/assets/游戏说明.txt)。

## 4. 视觉换皮

- [`styles.css`](../app/src/main/assets/styles.css)：`--text-color`、`--terminal-opacity`、字体
- 应用图标：`res/mipmap-*`
- 背景图：游戏内「背景图片设置」或默认资源

## 5. 打包发布

- 改包名：`app/build.gradle.kts` → `applicationId`
- 改签名：`keystore.properties`（勿提交 Git）
- 编译：见 [BUILD.md](BUILD.md)

## 6. 建议保留

- `seriesName: '文字修仙'` 或 README 中注明基于本框架（MIT 要求保留版权声明）
- 气运偏移与 Scheduler 架构（核心差异化）
- `meta-systems.js` 契约/成就框架，只换文案

## 7. 模块加载顺序

```
index.html
  → theme/config.js
  → theme/contracts.js
  → math.js
  → game.js
  → meta-systems.js
```

`meta-systems.js` 依赖 `game.js` 中的存档与 UI 函数，必须后加载。
